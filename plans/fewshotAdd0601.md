# 1️⃣ Lexical Agent（表层模式/耸动词/标点）

* **Input**: 只用标题
* **Focus**: 耸动词、标点、关键词
* **Prompt示例**：

```
你是词汇表层模式专家，专注于标题党模式、极端修饰、标点和耸动词判断。请判断以下标题是否属于点击诱导：

示例1：
标题：震惊！他居然用了这个方法
标签：class_1
解释：标题包含耸动词“震惊”和标点“！”
示例2：
标题：本市发布年度报告
标签：class_0
解释：普通新闻标题，没有耸动词或标点强调
待判断标题：
{title}
请严格输出JSON：
{
  "class_0_probability": float,
  "class_1_probability": float,
  "confidence": float,
  "explanation": "50字以内中文解释"
}
```

* **Few-shot选择**：

  * 3~5个标题，专注耸动词/标点/极端词
  * 尽量覆盖多种词汇组合和不同句式
  * 可以从真实新闻或点击诱饵标题中抽取

{title}

  "class_0_probability": float,

# 5️⃣ Consistency Agent（一致性/矛盾）

* **Input**: 标题 + 内容
* **Focus**: 标题与内容是否匹配，是否存在落差或转折
* **Prompt示例**：

```
你是一致性/矛盾专家，判断标题与内容是否一致，有无落差、转折或矛盾：

示例1：
标题：震惊！他在医院的秘密曝光
内容：医生发现男子在医院实验……
标签：class_1
解释：标题夸张，内容较中性，存在落差
示例2：
标题：市政府公布年度预算
内容：预算内容与标题一致
标签：class_0
解释：标题与内容一致
待判断标题：
{title}
内容：
{content}
严格输出JSON：
{
  "class_0_probability": float,
  "class_1_probability": float,
  "confidence": float,
  "explanation": "50字以内中文解释"
}
```

* **Few-shot选择**：

  * 选择标题与内容有落差或一致的对比案例
  * 可以用真实新闻或仿造新闻数据集
  * 3~5条即可

---

# 🔹 Few-shot选择原则

1. **真实案例优先**：标题和内容尽量来源真实新闻、论坛、微博、公开数据集
2. **覆盖Agent关注特征**：Lexical/Emotion/Intention → 标题特征；Semantic/Consistency → 标题+内容
3. **少量即可**：每个Agent 3~5条即可，保证Prompt简洁
4. **对比式**：最好包含正类（点击诱导）和负类（非点击诱导）
5. **避免冗余**：不要提供与Agent无关的特征，避免LLM分心
